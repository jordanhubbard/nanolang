# NanoLang RFCs

This directory contains RFCs (Requests for Comments) for NanoLang language evolution.

## What is an RFC?

An RFC is a formal proposal for a significant change to NanoLang. The RFC process ensures:
- Community input on major decisions
- Thorough design before implementation
- Documentation of design rationale
- Transparent decision-making

## Directory Structure

```
rfcs/
├── README.md              # This file
├── 0000-template.md       # Template for new RFCs
├── accepted/              # Accepted RFCs
├── rejected/              # Rejected RFCs (with reasons)
└── postponed/             # Postponed RFCs (good ideas, wrong time)
```

## How to Propose an RFC

See [RFC_PROCESS.md](../RFC_PROCESS.md) for complete guide.

**Quick start:**
1. Copy `0000-template.md` to `0000-my-feature.md`
2. Fill out template
3. Submit PR with title "RFC: My Feature"
4. Label PR with `T-rfc`
5. Engage with discussion
